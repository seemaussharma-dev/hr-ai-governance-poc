# 07 — AI Governance Plan

## Governance objective

Manage the system through its lifecycle using documented accountability, risk treatment, evidence, monitoring, and continual improvement.

## Lifecycle

| Phase | Required activity | Evidence |
|---|---|---|
| Idea | Define business problem and value | Business problem |
| Intake | Register system and owner | System inventory |
| Scope | Define intended and prohibited use | Scope document |
| Assess | Complete impact, privacy, and risk assessments | AIA, PIA, risk register |
| Design | Select controls and architecture | Control design, architecture |
| Build | Implement controls in code and process | Source code |
| Verify | Run functional and regulatory tests | Test report |
| Approve | Review residual risk and publication | Approval checklist |
| Operate | Monitor decisions and incidents | Audit events, metrics |
| Change | Assess code, policy, model, and scope changes | Change record |
| Retire | Remove access, logs, and obsolete artifacts | Retirement record |

## Governance principles

- Accountability
- Human oversight
- Transparency
- Privacy by design
- Security by design
- Purpose limitation
- Data minimization
- Traceability
- Robustness
- Controlled change
- Continual improvement

## Decision rights

- HR approves policy content.
- AI Governance approves intended use and risk acceptance.
- Privacy approves personal-data design.
- Security approves technical risk controls.
- Legal interprets regulatory applicability.
- Technical Lead implements and operates the prototype.

## Review frequency

- Before public publication.
- After any failed critical test.
- Before adding a model or real data.
- After a material incident.
- At least annually in a production implementation.

# 23 — Deployment and Operations

## Local deployment

1. Install Python.
2. Create virtual environment.
3. Install requirements.
4. Run tests.
5. Run Streamlit.
6. Use synthetic test questions.

## GitHub release

1. Run tests.
2. Search repository for secrets and real information.
3. Review `.gitignore`.
4. Review all synthetic documents.
5. Review disclaimers.
6. Create pull request.
7. Confirm GitHub Actions passes.
8. Merge and tag release.

## Operational roles

- Technical owner monitors application and tests.
- HR owner reviews policy content.
- AI governance owner reviews controls and residual risk.
- Privacy and security review incidents and material changes.

## Backup

The public repository is the source for code and synthetic documents. Do not use it to store real logs or employee information.

## Retirement

- Disable application.
- Archive final documentation.
- Remove audit logs according to retention.
- Revoke future API keys.
- Record retirement decision and unresolved risks.

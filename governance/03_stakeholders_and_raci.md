# 03 — Stakeholders and RACI

## Stakeholders

| Stakeholder | Interest |
|---|---|
| HR Director | Policy accuracy and employee experience |
| AI Governance Lead | Risk, controls, evidence, and approvals |
| Technical Lead | Architecture, code, testing, and operations |
| Privacy Lead or DPO | Personal-data controls and rights |
| Security Lead | Secure design, vulnerabilities, and incidents |
| Legal or Compliance | Regulatory interpretation |
| Employees | Clear, safe, useful answers |
| Internal Audit | Evidence and control effectiveness |

## RACI

| Activity | HR | AI Governance | Technical | Privacy | Security | Legal |
|---|---|---|---|---|---|---|
| Define business problem | A/R | C | C | C | I | C |
| Approve intended use | A | R | C | C | C | C |
| Approve policy documents | A/R | C | I | C | I | C |
| Design controls | C | A/R | R | R | R | C |
| Build prototype | I | C | A/R | C | C | I |
| Run tests | C | A | R | C | C | I |
| Approve publication | A | R | C | R | R | C |
| Handle incidents | A | R | R | R | R | C |
| Approve Stage 2 | A | R | C | R | R | C |

**R:** Responsible  
**A:** Accountable  
**C:** Consulted  
**I:** Informed

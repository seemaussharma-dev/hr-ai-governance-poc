# GitHub Publication Checklist

- [ ] All policies and company details are fictional.
- [ ] No employee names or personal data.
- [ ] No real company documents.
- [ ] No `.env` file.
- [ ] No API keys, tokens, passwords, or certificates.
- [ ] All tests pass.
- [ ] GitHub Actions is enabled.
- [ ] Limitations and disclaimer are visible.
- [ ] Regulatory mapping is described as educational.
- [ ] Repository issues warn users not to post personal data.
- [ ] License is present.
- [ ] README links to START_HERE.
